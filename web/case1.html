API Case Study: 1

A simple vulnerability in API leads to delete page of any users.

Case Study: 1

This case study involves a simple website similar to Indeed and Glassdoor.

Features of the website include:

  • Any company can register and post jobs for their organization.
  • Users can upload their documents, which admins can then review to forward or reject applications.

I registered as an organization and posted a job listing with details like job title, minimum experience, salary, etc. The job status remained pending.

While viewing the job, I noticed a numeric ID in the URL.

I sent the request to a repeater and attempted to change the ID to access other users' information, but received a 403 Forbidden error.

Then, I changed the request method from GET to DELETE.

And BOOM!!... the page was deleted.

I created a new account and replaced the new job ID with the old one from my first account. This allowed me to delete data using the DELETE method.

An attacker could brute-force job IDs and delete nearly all data.

In APIs, rate limiting is crucial to prevent such attacks.

Mitigation Strategies

To prevent similar vulnerabilities, consider the following measures:

  • Authorization Checks: Ensure proper authorization checks are in place to verify if the user has permission to delete the resource.
  • Rate Limiting: Implement rate limiting to prevent brute-force attacks.
  • Use HTTP Methods Properly: Only allow DELETE requests from authenticated and authorized users. Verify the request's legitimacy before processing it.
  • Input Validation: Validate and sanitize inputs to prevent unauthorized access.
  • Audit Logs: Maintain logs of API requests to detect and respond to suspicious activities.
  • Security Headers: Use security headers to protect against common attacks (e.g., X-Content-Type-Options, Content-Security-Policy).
  • Regular Security Testing: Conduct regular security testing, including penetration tests and code reviews, to identify and fix vulnerabilities.

Conclusion

API security is critical to protect sensitive data and maintain the integrity of your application. By implementing robust security measures, you can prevent unauthorized access and mitigate the risk of attacks.
